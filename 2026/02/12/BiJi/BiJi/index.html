Spring的笔记

IoC与DI

bean的命名规则：如果前两字母为大写就直接用类名，否则使用驼峰命名法。（可通过value指定名字，如：@Controller( value=”userController”)）

@Bean注解创建的bean，首先这个方法的类得拥有五大注解之一，方法名为bean名，通过name属性指定名

五大注解：@Component（所有的父类）、@Controller（控制层）、@Service（业务逻辑层）、@Repository（数据持久层）、@Configuration（配置类）

注入方法有属性注入、构造方法注入，setter方法注入。
他们的优缺点：

  • 属性注入：优点：简单方便
    缺点：只能用于IoC容器中，非IoC容器无法使用，且只在使用时才会触发空指针异常。无法注入final修饰的属性
  • 构造方法注入：优点：可以注入final修饰的属性。无法被修改。依赖的对象一定会被先初始化，因为加载使用这个对象一定得调用构造方法。通用性好
    缺点：注入多个对象时，代码会很繁琐
  • setter方法注入：优点：方便在实例化后，再次对注入对象进行修改。
    缺点：无法注入final修饰的属性，且维护麻烦，易被修改

@Autowired 与 @Resource 的区别：@Autowired是Spring框架提供的，而@Resource是JDK提供的。@Autowired是通过类型来进行寻找bean，而@Resource是通过名称寻找bean，并且有更多参数进行选择，如name来选择名字

Spring 与 Spring Boot 与 Spring MVC 的区别

  • Spring：是一个框架，是一种轻量型、一站式、模块化的框架，其目的是为了减化企业级应用程序开发。Spring的主要功能:管理对象，以及对象之间的依赖关系,⾯向切⾯编程,数据库事务管理,数据访问,web框架⽀持等
  • Spring Boot：Spring Boot是对Spring的⼀个封装,为了简化Spring应⽤的开发⽽出现的，中⼩型企业，没有成本研究⾃⼰的框架,使⽤SpringBoot可以更加快速的搭建框架,降级开发成本,让开发⼈员更加专注于Spring应⽤的开发，⽽⽆需过多关注XML的配置和⼀些底层的实现.
  • Spring MVC：SpringMVC是Spring的⼀个⼦框架,Spring诞⽣之后,⼤家觉得很好⽤,于是按照MVC模式设计了⼀个MVC框架(⼀些⽤Spring解耦的组件),主要⽤于开发WEB应⽤和⽹络接⼝，所以,Spring MVC是⼀个Web框架。
    最后⼀句话总结:SpringMVC和SpringBoot都属于Spring，SpringMVC是基于Spring的⼀个MVC框架，⽽SpringBoot是基于Spring的⼀套快速开发整合包。
