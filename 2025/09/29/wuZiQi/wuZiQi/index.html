五子棋项目详解

发表于2025-09-29|更新于2025-10-14|Java
|总字数:2.2k|阅读时长:9分钟|浏览量:

项目概述

本项目是一个基于WebSocket的在线五子棋对战平台，采用Spring Boot + MyBatis Plus架构，实现了实时对战、智能匹配、用户管理等功能。通过WebSocket技术实现低延迟的实时通信，为用户提供流畅的对战体验。

技术栈

  • 后端框架: Spring Boot 2.x, Spring MVC, Spring WebSocket
  • 数据持久层: MyBatis Plus, MySQL 8.0
  • 安全框架: Spring Security + JWT
  • 前端技术: HTML5, CSS3, JavaScript, WebSocket API
  • 构建工具: Maven
  • 其他: Lombok简化代码，Hibernate Validator参数校验

数据库设计

用户表（user_info）

字段名 类型 描述 备注
id BIGINT 主键 自增
username VARCHAR(50) 用户名 唯一索引
password VARCHAR(255) 密码 BCrypt加密
score INT 天梯分 默认1000
win_count INT 胜局数 默认0
total_count INT 总对局数 默认0
delete_flag TINYINT 删除标志 0-正常，1-删除
create_time DATETIME 创建时间 自动填充
update_time DATETIME 更新时间 自动更新

系统架构设计

项目源码：https://github.com/MIO-0701/wuZiQi

分层架构

  • Controller层: 负责接收前端请求，包括用户管理、匹配系统、游戏对战等模块的控制器
  • Service层: 核心业务逻辑实现层，处理用户认证、匹配算法、游戏状态管理等
  • Entity层: 实体类定义，对应数据库表结构
  • Mapper层: 数据访问层，使用MyBatis Plus简化CRUD操作
  • Util层: 工具类，包含对象转换、加密解密、常量定义等辅助功能
  • WebSocket层: 实时通信处理，管理游戏对战中的消息推送

核心模块

1. 用户管理模块

负责用户注册、登录认证、个人信息维护等功能

2. 匹配系统模块

基于天梯分的智能匹配算法，支持多段位匹配队列

3. 游戏对战模块

处理游戏逻辑、落子验证、胜负判定、悔棋等游戏核心功能

4. 实时通信模块

基于WebSocket的双向通信，确保低延迟的游戏体验

用户认证模块

用户注册

接口地址: POST /user/register

请求参数:

业务流程:

  1. 参数校验：验证用户名格式、密码强度等
  2. 唯一性检查：查询数据库确认用户名是否已存在
  3. 密码加密：使用BCrypt算法对密码进行加密存储
  4. 初始化用户：设置默认天梯分1000，胜负场次为0
  5. 数据入库：将用户信息插入数据库
  6. 返回结果：成功返回用户ID，失败返回错误信息

安全考虑:

  • 密码使用BCrypt加密，不可逆且带盐值
  • 用户名唯一索引防止重复注册
  • 参数校验防止SQL注入

接口地址: POST /user/login

业务流程:

  1. 根据用户名查询用户信息
  2. 使用BCrypt校验密码是否正确
  3. 生成JWT令牌包含用户ID和用户名
  4. 将用户信息存入Redis缓存，设置过期时间
  5. 返回JWT令牌和用户基本信息

智能匹配系统

WebSocket连接建立

连接地址: ws://localhost:8080/match

匹配流程:

  1. 前端建立WebSocket连接时携带JWT令牌进行身份验证
  2. 后端验证用户身份，防止未登录用户匹配
  3. 根据用户天梯分将用户放入相应分段的匹配队列

分段匹配策略

匹配状态管理

  • 匹配中: 用户进入匹配队列，开始计时
  • 匹配成功: 找到对手，创建游戏房间
  • 取消匹配: 用户主动取消，移出队列
  • 匹配超时: 超过5分钟未匹配成功，自动取消

房间创建

匹配成功后，系统会：

  1. 创建唯一的房间ID
  2. 将两位玩家加入房间
  3. 初始化游戏状态
  4. 通知双方匹配成功和游戏房间信息

游戏初始化

  1. 房间分配: 玩家进入指定房间，系统确认双方就位
  2. 先手决定: 随机决定先手方（黑棋）和后手方（白棋）
  3. 状态同步: 向双方发送游戏开始消息，包含对手信息和先后手状态

项目亮点

  1. 实时性强: 基于WebSocket实现毫秒级延迟的实时对战
  2. 匹配智能: 分段位匹配算法，确保公平竞技
  3. 安全可靠: JWT认证 + BCrypt加密，保障用户数据安全
  4. 扩展性好: 模块化设计，便于后续功能扩展

部署说明

  1. 安装MySQL数据库，执行项目中的SQL脚本创建表结构
  2. 配置application.yml中的数据库连接信息
  3. 执行mvn clean package编译打包
  4. 运行java -jar wuZiQi-1.0.0.jar启动服务
  5. 前端页面部署到Nginx或Tomcat即可访问
